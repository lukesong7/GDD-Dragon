# UI/UX 설계 (UI/UX Design)

- **프로젝트명**: Dragon
- **장르**: SF 리니지 라이크 MMORPG
- **버전**: 2.0
- **최종 수정일**: 2026-03-23

---

## 1. UI 설계 원칙

| 원칙 | 설명 |
|------|------|
| **정보 우선순위** | 전투 중 필수 정보만 홀로그래픽 HUD에 상시 표시, 나머지는 접근 가능하되 숨김 |
| **원클릭 접근** | 모든 주요 기능은 최대 2클릭 이내 접근 |
| **플랫폼별 최적화** | PC와 모바일 UI를 완전 분리 설계 |
| **커스터마이징** | HUD 위치, 크기, 투명도를 유저가 자유 조정 |
| **색상 일관성** | 아이템 등급 색상, 상태 색상을 게임 전체에 일관 적용 (홀로그램 UI 톤 통일) |

## 3. 모바일 UI 레이아웃

```
┌─────────────────────────────────────┐
│ [미니맵]  [타겟HP]  [시스템메뉴]    │
│                                      │
│ [파티원]     [게임 화면]             │
│                                      │
│                           [스킬버튼] │
│ [채팅]                    [4개]      │
│         [가상패드]        [기본공격] │
│ [HP/EP]            [메디키트][자동전투]│
│ [경험치바]                           │
└─────────────────────────────────────┘
```

### 3.1 모바일 전용 UI 요소

| 요소 | 설명 |
|------|------|
| 가상 조이스틱 | 좌측 하단, 이동 |
| 스킬 버튼 | 우측 4~6개 원형 버튼, 쿨다운 표시 |
| 기본 공격 | 우측 하단 대형 버튼 |
| 자동 전투 | 토글 버튼, 눈에 잘 띄는 위치 |
| 타겟팅 보조 | 가장 가까운 적 자동 선택 + 좌우 스와이프로 타겟 전환 |
| 퀵 메뉴 | 화면 상단 드래그로 펼치기 |

> **[디자이너 의견]** 모바일에서 스킬 버튼이 6개를 넘으면 손가락이 겹치고 오조작이 빈발한다. 기본 배치 4개 + 확장 시 최대 6개가 한계. 나머지 스킬은 자동 전투 시 AI가 사용하도록 위임. UI 전체 톤은 반투명 홀로그래픽 블루/시안 계열로 통일하여 SF 세계관 몰입을 유지한다.
>
> **[프로그래머 의견]** 모바일 가상 조이스틱은 터치 데드존 처리가 중요하다. 너무 민감하면 의도치 않은 이동, 너무 둔하면 반응 없음으로 느껴진다. 데드존 반경과 감도를 유저가 설정 가능하게 해야 한다.
>
> **[기획자 결정]** 모두 수용. 모바일 스킬 버튼은 기본 4개. 설정에서 6개까지 확장 가능. 조이스틱 감도는 설정 > 조작에서 3단계(낮음/보통/높음) + 커스텀 슬라이더 제공. 홀로그래픽 UI 톤은 디자인 가이드로 문서화하여 전체 UI에 일관 적용.

## 4. 주요 UI 창

### 4.1 UI 창 목록

| 창 | 단축키 (PC) | 핵심 정보 |
|----|-------------|-----------|
| 캐릭터 정보 | C | 스탯, 장비, 전투력 |
| 인벤토리 | I | 소지품, 무게 |
| 스킬 창 | K | 스킬 트리, SP |
| 퀘스트 | Q | 진행 중/완료 퀘스트 |
| 지도 | M | 행성맵, 영토 현황 |
| 군단 | G | 군단 정보, 군단원 |
| 거래소 | 없음 (데이터 터미널) | 아이템 검색/등록 |
| 시스템 설정 | ESC | 그래픽, 사운드, 조작 |
| 랭킹 | R | 개인/군단/PvP 랭킹 |
| 펫 | P | 펫 정보, 스킬, 성장 |

### 4.2 거래소 UI

```
┌──────────────────────────────────────┐
│ [검색바]  [카테고리 필터]  [등급 필터] │
│ [옵션 필터: 스탯 종류, 최소값]        │
│──────────────────────────────────────│
│ 아이템명  |  옵션  |  가격  |  판매자  │
│ ─────────|─────── |─────── |──────── │
│ (아이템 목록, 홀로그래픽 스크롤)       │
│──────────────────────────────────────│
│ [선택 아이템 상세]  [시세 그래프]       │
│ [구매]  [즐겨찾기]                     │
└──────────────────────────────────────┘
```

## 5. 조작 설정

### 5.1 PC 기본 조작

| 조작 | 키 |
|------|-----|
| 이동 | WASD 또는 마우스 클릭 |
| 일반 공격 | 좌클릭 (타겟 지정 후) |
| 스킬 | 1~= 키, Shift+1~= |
| 타겟 전환 | Tab |
| 줍기 | F |
| 인터랙션 | E |
| 자동 달리기 | Num Lock |
| PK 모드 토글 | Ctrl+P |

### 5.2 키 바인딩

- 모든 단축키 커스터마이징 가능
- 프리셋 3개 저장/로드

## 6. 채팅 시스템

### 6.1 채팅 채널

| 채널 | 범위 | 색상 |
|------|------|------|
| 일반 | 반경 100m | 흰색 |
| 외침 | 반경 500m | 노란색 |
| 거래 | 서버 전체 | 녹색 |
| 파티 | 파티원 | 파란색 |
| 군단 | 군단원 | 하늘색 |
| 동맹 | 동맹 군단원 | 보라색 |
| 귓속말 | 1:1 | 분홍색 |
| 시스템 | 전체 | 주황색 |

### 6.2 채팅 필터

- 욕설 필터 (서버 설정)
- 스팸 방지: 같은 메시지 5초 내 재전송 불가
- 거래 채널: Lv.20 이상만 사용 가능

> **[프로그래머 의견]** 거래 채널 스팸은 모든 MMORPG의 고질적 문제다. Lv.20 제한 외에도, 거래 채널 발언 쿨다운(30초)과 광고/RMT 패턴 자동 감지 시스템을 추가해야 한다.
>
> **[기획자 결정]** 수용. 거래 채널 발언 쿨다운 30초, AI 기반 RMT 광고 자동 뮤트 시스템 적용.
