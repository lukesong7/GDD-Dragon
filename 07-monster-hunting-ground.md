# 호스틸 & 작전 구역 시스템 (Hostiles & Operation Zones)

- **프로젝트명**: Dragon
- **버전**: 2.0
- **장르**: SF 리니지 라이크 MMORPG
- **최종 수정일**: 2026-03-23

## 1. 호스틸 분류

### 1.1 등급별 분류

| 등급 | 표시 | 체력 배율 | 출현 방식 | 전투 인원 |
|------|------|-----------|-----------|-----------|
| **일반 (Normal)** | 이름만 표시 | ×1 | 상시 리젠 | 솔로 |
| **강화 (Enhanced)** | 이름 + ★ | ×3 | 상시 리젠 (소수) | 솔로~2인 |
| **정예 (Elite)** | 금색 이름 | ×10 | 일정 주기 리젠 | 분대 (4~8인) |
| **필드 보스 (Field Boss)** | 붉은 이름 + 아이콘 | ×50 | 1~4시간 리젠 | 다수 분대 |
| **레이드 보스 (Raid Boss)** | 특수 UI | ×200 | 정해진 조건 | 레이드 (16~32인) |
| **드래곤 AI** | 드래곤 아이콘 | ×30~100 | 이벤트/조건 | 분대~레이드 |

### 1.2 호스틸 유형별 분류

| 유형 | 설명 | 주요 출현지 |
|------|------|------------|
| **기계수 (Machine Beast)** | 폭주한 바이오-메카 생체병기 | 야외 작전 구역, 폐기된 연구시설 |
| **로그 드론 (Rogue Drone)** | 통제를 벗어난 자율 전투 드론 | 폐기지, 군사 시설 |
| **변이체 (Mutant)** | 에너지 오염으로 변이된 토착 생물 | 오염 구역, 방사능 지대 |
| **부패 AI 구조물 (Corrupted AI)** | 드래곤 AI에 감염된 시설/병기 | 고레벨 작전 구역, 드래곤 코어 지대 |
| **적대 세력 병사 (Hostile NPC)** | 적대 팩션 소속 NPC 전투원 | 분쟁 구역, 기지 주변 |
| **드래곤 AI 개체** | 자아를 가진 고대 기계 생명체 | 드래곤 코어 지대 전용 |

### 1.3 속성별 분류

모든 호스틸은 4대 속성 + 무속성 중 하나를 보유:

| 속성 | 주요 서식지 | 특수 공격 패턴 |
|------|------------|---------------|
| 플라즈마 (열) | 이그니스 용광 지대, 코어 반응로 | 플라즈마 연소 DoT, 범위 폭발 |
| 냉각 (냉) | 글래시아 동토 기지, 냉각 시설 | 냉각제 유출, 구동 제한 |
| 전기 (전자) | 테슬라 폐발전소, 전자전 구역 | EMP, 시스템 과부하, 쉴드 붕괴 |
| 중력 (파동) | 중력 이상 지대, 워프 균열 | 중력파 속박, 넉백, 공간 왜곡 |
| 무속성 | 범용 | 물리 타격, 일반 사격 |

## 2. 호스틸 AI

### 2.1 AI 등급

| AI 등급 | 적용 대상 | 행동 패턴 |
|---------|-----------|-----------|
| **단순** | 일반 호스틸 | 감지 → 접근 → 공격 반복 |
| **기본** | 강화 호스틸 | 단순 + HP 낮으면 도주/방어 모드 전환 |
| **전술** | 정예 호스틸 | 어그로 관리, 스킬 패턴 사용, 취약 대상(메딕/테크노서) 우선 공격 |
| **고급** | 필드/레이드 보스 | 페이즈 전환, 다수 스킬 패턴, 기믹 보유 |
| **특수** | 드래곤 AI 개체 | 고급 + 환경 파괴, 속성 변환, 적응형 학습 |

### 2.2 보스 패턴 설계 원칙

- **페이즈 시스템**: HP 구간별 행동 패턴 전환 (100~70%, 70~30%, 30~0%)
- **텔레그래프**: 강력한 공격 전 반드시 시각적 예고 (홀로그래픽 경고 표시, 1~2초 선딜)
- **기믹**: 단순 딜만으로 클리어 불가한 요소 (예: 특정 콘솔 해킹, 전력 차단, 코어 파괴)
- **소환**: 보스가 주기적으로 드론/졸개 소환, 분대 역할 분담 유도

### 2.3 보스 예시: 코어 가디언 Mk.VII (레이드 보스)

| 페이즈 | HP 구간 | 주요 패턴 |
|--------|---------|-----------|
| 1 | 100~70% | 레일건 포격(전방 직선), 드론 소환(4기), 에너지 배리어 생성 |
| 2 | 70~30% | 플라즈마 브레스(원뿔 범위), 중력장 생성(바닥 기믹), 쉴드 붕괴 EMP |
| 3 | 30~0% | 오버드라이브 모드(전 능력 +50%), 궤도 폭격 소환, 자폭 카운트다운 기믹 |

> **[프로그래머 의견]** 보스 AI를 상태 머신(State Machine)이 아닌 행동 트리(Behavior Tree)로 구현하면 복잡한 패턴 조합이 훨씬 유연하다. 기획자가 비주얼 에디터에서 직접 보스 패턴을 조합할 수 있도록 툴을 만들면 개발 속도가 크게 향상된다.
>
> **[디자이너 의견]** SF 보스는 기계적 변형 연출이 핵심이다. 페이즈 전환 시 파워아머/외형이 물리적으로 변형되는 연출(부위 분리, 추가 무장 전개, 코어 노출 등)이 필요하다. 각 페이즈별 BGM도 전환되어야 전투 몰입도가 유지된다.
>
> **[기획자 결정]** 수용. 보스 AI 비주얼 에디터는 개발 파이프라인 초기에 구축한다. 페이즈 전환 연출은 1~2초 컷신(스킵 가능)으로 처리하되, 이 동안 보스는 무적 상태.

## 3. 작전 구역 시스템

### 3.1 작전 구역 설계 원칙

| 원칙 | 설명 |
|------|------|
| **레벨 구간별 배치** | 각 레벨 구간에 2~3개 소탕 구역 선택지 |
| **효율 차등** | 높은 효율 = 높은 위험 (PK 위험, 강력한 호스틸) |
| **병목 설계** | 의도적으로 최적 소탕 구역을 제한 → 경쟁 유발 |
| **경로 다양성** | 같은 레벨이라도 안전/위험 루트 제공 |

### 3.2 작전 구역 목록 (행성 아르카디아)

| 작전 구역 | 레벨 | 밀집도 | PK 위험도 | 핵심 드롭 |
|-----------|------|--------|-----------|-----------|
| 루미나 안전 구역 | 1~10 | 낮음 | 안전지대 | 기본 장비, 소비 |
| 폐기 드론 집결지 | 8~15 | 중간 | 낮음 | 고급 합금 재료 |
| 은빛 숲 외곽 초소 | 12~20 | 중간 | 낮음 | 나노 섬유 재료 |
| 은빛 숲 심층 시설 | 18~25 | 높음 | 중간 | 희귀 나노 물질, 경장갑 파워아머 |
| 폐광산 채굴 시설 | 22~32 | 높음 | 중간 | 희귀 광석, 무기공방 설계도 |
| 아덴 평원 북부 기지 잔해 | 30~40 | 중간 | 높음 | 영웅 장비 재료 |
| 아덴 평원 남부 격전지 | 35~45 | 높음 | 높음 | 영웅 장비 드롭 |
| 테슬라 폐발전소 입구 | 40~50 | 높음 | 매우 높음 | 고급 에너지 결정 |
| 테슬라 폐발전소 심부 | 48~58 | 매우 높음 | 매우 높음 | 전설 재료 |
| 코어 반응로 외곽 | 50~60 | 중간 | 높음 | 드래곤 코어(소) |
| 코어 반응로 중심부 | 58~68 | 높음 | 매우 높음 | 드래곤 코어(중) |
| 그림자 늪 오염 구역 | 55~65 | 중간 | 높음 | 중력 속성 재료, 에이전트 장비 |
| 폐허의 전장 분쟁 지대 | 60~72 | 매우 높음 | 최고 | 전설 장비, PvP 보상 |
| 드래곤 코어 지대 외곽 | 68~80 | 높음 | 최고 | 드래곤 코어(대) |
| 드래곤 코어 지대 핵심 | 78~90 | 매우 높음 | 최고 | 신화 재료, 전설 프로그램 모듈 |

### 3.3 작전 구역 경쟁 메카닉

> **[기획자 의견]** 소탕 구역 경쟁은 리니지 라이크의 핵심이다. 최적 소탕 구역을 '부족하게' 설계하여 군단 간 갈등을 자연스럽게 유발하는 것이 핵심이다.

- **채널 시스템**: 작전 구역당 3~5개 채널, 채널 이동 쿨다운 10분
- **점유 보너스**: 같은 소탕 구역에서 30분 이상 연속 소탕 시 드롭률 +5%
- **경쟁 알림**: 다른 군단원이 같은 작전 구역 진입 시 알림
- **영토 작전 구역**: 군단이 점령한 구역은 점령 군단에 드롭률 +10%, 비점령 군단 -5%

## 4. 드롭 테이블

### 4.1 드롭 구조

```
호스틸 처치
├── 확정 드롭: 크레딧 (100%)
├── 일반 드롭 테이블: 소비/재료 (등급별 확률)
├── 장비 드롭 테이블: 무기/파워아머 (등급별 확률)
└── 특수 드롭 테이블: 프로그램 모듈/드래곤 코어/이벤트 아이템 (극저확률)
```

### 4.2 드롭 확률 기준 (일반 호스틸)

| 드롭 종류 | 확률 |
|-----------|------|
| 크레딧 | 100% (호스틸 레벨에 비례) |
| 일반 소비/재료 | 30~50% |
| 고급 재료 | 5~10% |
| 일반 장비 | 3~5% |
| 고급 장비 | 0.5~1% |
| 희귀 장비 | 0.05~0.1% |
| 영웅 장비 | 0.001% |

- 정예 호스틸: 일반 대비 드롭 확률 ×5, 영웅 장비 드롭 가능
- 필드 보스: 영웅 장비 확정 1개 + 전설 재료 확률 드롭
- 레이드 보스: 전설 장비 확정 1~2개 + 신화 재료 확률 드롭

### 4.3 드롭 권한

| 방식 | 적용 상황 |
|------|-----------|
| **최다 기여** | 필드 보스 — 기여도가 가장 높은 분대 |
| **분대 분배** | 분대 소탕 — 설정에 따라 |
| **레이드 분배** | 레이드 보스 — 레이드 지휘관 분배 또는 주사위 |
| **개인 드롭** | 일반 호스틸 — 마지막 타격자 |

> **[프로그래머 의견]** 필드 보스에서 '최다 피해' 방식은 메딕이 불리하다. 데미지 기여뿐 아니라 힐/버프 기여도를 포함한 '기여도 점수' 시스템으로 확장해야 메딕이 분대에서 환영받는다.
>
> **[기획자 결정]** 수용. 기여도 = 딜링(60%) + 힐링(25%) + 탱킹(15%). 기여도가 가장 높은 분대에 드롭 권한 부여.

## 5. 리젠 시스템

| 호스틸 등급 | 리젠 시간 | 특이사항 |
|------------|----------|----------|
| 일반 | 30~60초 | 채널별 독립 |
| 강화 | 1~3분 | |
| 정예 | 10~30분 | 출현 시 지역 공지 |
| 필드 보스 | 1~4시간 (±30분 랜덤) | 서버 전체 공지 |
| 레이드 보스 | 고정 스케줄 또는 조건 트리거 | 서버 전체 공지 + 사전 연출 |
| 드래곤 AI 개체 | 동적 이벤트 트리거 | 서버 전체 경보 + 사이렌 연출 |

> **[프로그래머 의견]** 필드 보스 리젠 시간에 랜덤 편차를 주는 것은 '보스 캠핑'을 완전히 방지하지는 못하지만, 캠핑의 비용(시간)을 높여 건전한 경쟁을 유도한다. 리젠 타이머는 서버 내부에서만 관리하고 클라이언트에 절대 노출하지 않아야 한다.
>
> **[디자이너 의견]** 드래곤 AI 개체 출현 시 행성 전역에 경보 사이렌 + 하늘 색상 변화(붉은 오로라) 연출이 필요하다. 필드 보스 출현은 해당 구역 지면에 에너지 균열이 나타나며 기계수가 조립되는 연출을 적용한다.
>
> **[기획자 결정]** 수용. 추가로 필드 보스가 일정 시간 내 처치되지 않으면 자동 소멸(리젠 타이머 리셋)하여 한 세력이 보스를 '잠금(locking)'하는 행위를 방지한다.

## 6. 호스틸 도감

### 6.1 도감 시스템

- 처치한 호스틸은 자동으로 도감에 등록
- 도감 등록 시 해당 호스틸의 약점 속성, 주요 드롭, 출현 위치 정보 공개
- 도감 완성도에 따라 영구 보너스 제공:

| 도감 완성도 | 보너스 |
|------------|--------|
| 25% | 호스틸 대상 데미지 +2% |
| 50% | 호스틸 드롭률 +3% |
| 75% | 호스틸 대상 데미지 +3% + 방어력 +2% |
| 100% | 칭호 "아르카디아의 사냥꾼" + 전 보너스 합산 + 특수 외형 |

### 6.2 스캔 시스템

- 전투 중 호스틸을 타겟팅하면 자동 스캔 진행 (3초)
- 스캔 완료 시 해당 개체의 HP, 속성, 약점이 HUD에 표시
- 레인저 클래스: 스캔 시간 50% 감소, 추가 정보(스킬 패턴) 표시

> **[기획자 의견]** 스캔 시스템은 SF 테마에 자연스럽게 녹아드는 정보 제공 수단이다. 도감과 연계하여 '정보 수집'이라는 게임 내 활동을 부여하고, 레인저 클래스에 정찰 역할의 가치를 추가한다.
>
> **[프로그래머 의견]** 스캔 데이터는 클라이언트 캐시로 관리하고, 도감 완성도만 서버에 저장하면 된다. 스캔 정보는 분대원과 공유 가능하도록 네트워크 동기화가 필요하다.
>
> **[기획자 결정]** 수용. 스캔 정보는 분대 채팅으로 공유 가능하며, 레인저의 스캔은 분대원 전원에게 자동 공유.
