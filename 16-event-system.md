## 1. 출석 보상

### 1.1 일일 출석

| 일수 | 보상 |
|------|------|
| 1일 | HP 메디키트(대) ×20 |
| 3일 | 경험치 부스트 데이터칩 (1시간) |
| 7일 | 고급 강화 모듈 ×3 |
| 14일 | 영웅 장비 컨테이너 |
| 21일 | 축복 프로토콜 ×2 |
| 28일 (개근) | 영웅 장비 선택 컨테이너 + 전용 칭호 |

- 출석은 접속 시 자동 체크 (하루 1회)
- 놓친 날은 복구 불가 (프리미엄 패스: 3일 복구 가능)

### 1.2 복귀 유저 보상

| 조건 | 보상 |
|------|------|
| 7일 이상 미접속 후 복귀 | 7일간 경험치 ×2 버프 |
| 30일 이상 미접속 후 복귀 | 14일간 경험치 ×2 + 복귀 장비 세트 + 크레딧 500,000 |

## 2. 이벤트 시스템

### 2.1 정기 이벤트

| 이벤트 | 주기 | 내용 |
|--------|------|------|
| **드래곤 프로토콜 페스티벌** | 분기 (3개월) | 서버 전체 축제, 한정 아머 스킨, 미니게임 |
| **군단 대전** | 월간 | 군단 토너먼트, 랭킹 보상 |
| **시즌 전장** | 3개월 시즌 | PvP 랭킹 정산, 시즌 보상 |
| **드래곤 강림** | 비정기 (월 1~2회) | 대형 드래곤 월드 보스 이벤트 |
| **주말 작전 축제** | 매주 토~일 | 경험치/드롭률 +20% |

### 2.2 시즌 이벤트 (현실 연동)

| 시기 | 이벤트 | 콘텐츠 |
|------|--------|--------|
| 설/추석 | 명절 이벤트 | 한정 전통 아머 스킨, 떡/송편 나노 보급품 |
| 여름 | 행성 해변 작전 | 하이드로 아머 스킨, 수상 기지 맵 |
| 할로윈 | 바이러스 프로토콜 | 감염구역 던전, 변이체 변신 아이템 |
| 크리스마스 | 동계 보급 작전 | 극지 아머 스킨, 눈 내리는 작전 구역 |
| 기념일 | 서버 오픈 기념 | 기념 칭호, 대규모 보상 |

### 2.3 동적 서버 이벤트

01-game-concept에서 정의한 '동적 월드 이벤트'와 연계:

| 트리거 | 이벤트 | 보상 |
|--------|--------|------|
| 특정 작전 구역 누적 10,000킬 | 기계수 대규모 침공 | 드래곤 코어, 영웅 장비 |

| 기지 쟁탈전 3연승 군단 발생 | 도전 세력 버프 | 공격측 공격력 +10% |
| 서버 전체 레벨 평균 상승 | 신규 작전 구역 개방 | 새로운 콘텐츠 해금 |

## 3. 이벤트 운영 원칙

| 원칙 | 설명 |
|------|------|
| **FOMO 최소화** | 한정 아이템은 외형만, 전투력 영향 아이템은 재등장 |
| **참여 용이성** | 이벤트 참여 조건은 낮게, 최대 보상은 활동량에 비례 |
| **서버 활성화** | 이벤트 시간대에 접속 인원이 집중되도록 설계 |
| **신규 유저 배려** | 모든 이벤트에 저레벨 참여 경로 제공 |

> **[기획자 의견]** 이벤트는 '놓치면 영영 사라지는 전투력'이 아닌, '놓치면 아쉬운 외형/칭호' 수준이어야 한다. FOMO로 단기 접속을 끌어올릴 수 있지만, 장기적으로는 유저 피로와 이탈을 유발한다. 특히 SF 세계관의 시즌 이벤트는 현실 명절과 연동하되, 행성 식민지 축제로 자연스럽게 치환하여 몰입을 유지한다.
>
> **[프로그래머 의견]** 이벤트 시스템은 코드 수정 없이 운영팀이 직접 설정할 수 있는 관리 툴이 필수. 이벤트 기간, 보상, 조건을 DB 기반으로 관리하고, 핫픽스 없이 라이브에 적용 가능해야 한다. 동적 서버 이벤트의 트리거는 실시간 집계 시스템과 연동해야 하므로 초기 아키텍처 설계에 반영 필요.
>
> **[기획자 결정]** 수용. 이벤트 관리 시스템은 운영 툴의 핵심 기능으로 개발 우선순위를 높인다. 동적 이벤트 트리거 집계 모듈은 서버 인프라 설계 단계에서 함께 설계한다.
