# 아이템 시스템 (Item System)

- **프로젝트명**: Dragon
- **버전**: 1.0
- **최종 수정일**: 2026-03-23

---

## 1. 아이템 분류

### 1.1 대분류

| 분류 | 설명 | 거래 가능 여부 |
|------|------|----------------|
| **장비** | 무기, 방어구, 액세서리 | 대부분 가능 (일부 귀속) |
| **소비** | 물약, 주문서, 음식 | 가능 |
| **재료** | 제작/강화 재료 | 가능 |
| **퀘스트** | 퀘스트 전용 아이템 | 불가 |
| **기타** | 텔레포트 스크롤, 귀환석 등 | 가능 |

### 1.2 아이템 등급

| 등급 | 색상 | 드롭 확률 (일반 몬스터) | 특징 |
|------|------|----------------------|------|
| **일반 (Normal)** | 흰색 | 높음 | 기본 스탯만 존재 |
| **고급 (Uncommon)** | 녹색 | 중간 | 추가 옵션 1개 |
| **희귀 (Rare)** | 파란색 | 낮음 | 추가 옵션 2개 |
| **영웅 (Hero)** | 보라색 | 매우 낮음 | 추가 옵션 3개 + 세트 효과 |
| **전설 (Legend)** | 주황색 | 보스/레이드 전용 | 추가 옵션 4개 + 고유 효과 |
| **신화 (Mythic)** | 붉은색 | 제작 전용 | 추가 옵션 5개 + 고유 효과 + 변신 외형 |

> **[디자이너 의견]** 등급별 색상은 업계 표준을 따르되, 아이템 아이콘 테두리와 이름 색상을 동시에 적용해야 인벤토리에서 한눈에 구분된다. 전설 이상은 아이콘에 미세한 반짝임 애니메이션을 추가하여 수집 욕구를 자극해야 한다.
>
> **[기획자 결정]** 수용. 추가로 신화 등급은 획득 시 서버 전체 공지를 띄워 희소성을 부각한다.

## 2. 장비 시스템

### 2.1 장비 슬롯

| 슬롯 | 위치 | 비고 |
|------|------|------|
| 무기 (주무기) | 오른손 | 직업별 무기 제한 |
| 보조 무기/방패 | 왼손 | 방패/보조무기/없음 |
| 투구 | 머리 | |
| 상의 | 상체 | |
| 하의 | 하체 | |
| 장갑 | 손 | |
| 신발 | 발 | |
| 망토 | 등 | |
| 목걸이 | 목 | |
| 귀걸이 | 귀 | 2개 슬롯 |
| 반지 | 손가락 | 2개 슬롯 |
| 벨트 | 허리 | 물약 자동 사용 슬롯 연동 |

### 2.2 장비 옵션

#### 기본 옵션 (등급 무관, 모든 장비 보유)
- 무기: 공격력, 공격 속도
- 방어구: 방어력, HP 보너스
- 액세서리: 스탯 보너스

#### 추가 옵션 (랜덤 부여)

| 옵션 | 수치 범위 (영웅 등급 기준) |
|------|--------------------------|
| STR/DEX/CON/INT/WIS/LUK | +3~10 |
| 크리티컬 확률 | +1~5% |
| 크리티컬 데미지 | +5~20% |
| 공격 속도 | +3~10% |
| 이동 속도 | +3~8% |
| HP/MP 추가 | +100~500 |
| PvP 공격력/방어력 | +2~8% |
| 속성 공격/저항 | +5~15 |
| 드롭률 증가 | +1~5% |
| 경험치 증가 | +1~5% |

> **[프로그래머 의견]** 추가 옵션이 완전 랜덤이면 '최적 옵션 조합'을 위한 무한 파밍이 발생한다. 옵션 재설정(리롤) 시스템을 제공하되, 재화 소모를 통해 경제 소멸 장치로 활용하면 좋다.
>
> **[기획자 결정]** 수용. '축복의 오브' 아이템으로 추가 옵션 1개를 랜덤 재설정 가능. '상급 축복의 오브'로 원하는 옵션 카테고리를 지정하고 수치만 랜덤. 둘 다 제작/거래 가능.

### 2.3 세트 장비

영웅 등급 이상 장비는 세트를 구성할 수 있다.

| 세트 피스 수 | 보너스 예시 |
|-------------|-----------|
| 2개 | 스탯 +5 |
| 4개 | 고유 효과 (예: 피격 시 10% 확률 보호막) |
| 6개 (풀세트) | 극강 효과 (예: 스킬 쿨다운 -15%) |

## 3. 강화 시스템

### 3.1 일반 강화 (+1 ~ +20)

| 강화 단계 | 성공 확률 | 실패 시 | 효과 |
|-----------|----------|---------|------|
| +1 ~ +3 | 100% | - | 기본 스탯 +3%/단계 |
| +4 ~ +6 | 80% | 강화 수치 유지 | 기본 스탯 +4%/단계 |
| +7 ~ +9 | 50% | 강화 -1 | 기본 스탯 +5%/단계 + 글로우 이펙트 |
| +10 ~ +12 | 30% | 강화 -2 | 기본 스탯 +6%/단계 + 오라 이펙트 |
| +13 ~ +15 | 15% | 강화 -3 또는 장비 파괴(10%) | 기본 스탯 +8%/단계 + 특수 이펙트 |
| +16 ~ +20 | 5~10% | 강화 -3 또는 장비 파괴(30%) | 기본 스탯 +10%/단계 + 최상위 이펙트 |

### 3.2 강화 보호 아이템

| 아이템 | 효과 | 획득처 |
|--------|------|--------|
| 축복 주문서 | 실패 시 강화 수치 유지 | 제작, 캐시샵 |
| 보호 주문서 | 실패 시 장비 파괴 방지 | 이벤트, 캐시샵 |
| 확률 증가 주문서 | 성공 확률 +5~10% | 제작, 이벤트 |

> **[프로그래머 의견]** 강화 시스템은 서버에서만 확률 판정을 수행해야 한다. 클라이언트는 연출만 담당. 확률 판정 로그는 전수 기록하여 운영 툴에서 조회 가능하게 해야 CS 대응이 가능하다.
>
> **[디자이너 의견]** 강화 연출이 매우 중요하다. +10 이상 시도 시 카메라가 줌인되고, 성공하면 빛이 폭발하며, 실패하면 장비가 흔들리는 연출. 파괴 시에는 장비가 부서지는 연출이 필요하다. 이 연출이 강화의 도박적 쾌감을 만든다.
>
> **[기획자 결정]** 모두 수용. 강화 연출은 단계별로 긴장감이 점증하도록 설계하며, +15 이상 성공 시 서버 전체 공지를 띄운다.

### 3.3 속성 부여

- 무기에 5대 속성(화/빙/지/풍/암) 중 하나를 부여 가능
- **드래곤 에센스** 소모
- 속성 상성에 따라 대상에 추가 데미지 또는 감소

## 4. 제작 시스템

### 4.1 제작 카테고리

| 카테고리 | 제작 가능 아이템 | 핵심 재료 |
|----------|-----------------|-----------|
| **대장간 (무기/방어구)** | 영웅~신화 등급 장비 | 광석, 가죽, 드래곤 에센스 |
| **연금술 (소비)** | 물약, 주문서, 축복 오브 | 약초, 마나석, 몬스터 부산물 |
| **보석 세공 (액세서리)** | 반지, 목걸이, 귀걸이 | 원석, 드래곤 에센스 |

### 4.2 제작 성공률

| 등급 | 성공률 | 실패 시 |
|------|--------|---------|
| 영웅 | 60% | 재료 50% 반환 |
| 전설 | 30% | 재료 30% 반환 |
| 신화 | 10% | 재료 20% 반환 |

- 드워프 종족: 제작 성공률 +10%
- 제작 숙련도: 반복 제작으로 성공률 점진적 증가 (최대 +15%)

> **[기획자 의견]** 신화 등급 장비는 제작으로만 획득 가능하다. 이는 드워프 종족과 제작 전문 유저에게 경제적 가치를 부여하고, 제작 콘텐츠 자체를 엔드게임 활동으로 만드는 장치다.

## 5. 인벤토리 및 창고

### 5.1 인벤토리

| 항목 | 수치 |
|------|------|
| 기본 슬롯 | 40칸 |
| 최대 확장 | 100칸 (캐시샵 확장권) |
| 무게 시스템 | 있음 (STR에 따라 최대 무게 증가) |
| 무게 초과 시 | 이동 속도 감소, 일정 이상 이동 불가 |

### 5.2 창고

| 항목 | 수치 |
|------|------|
| 개인 창고 | 100칸 (마을 NPC 이용) |
| 혈맹 창고 | 200칸 (권한별 접근) |
| 창고 확장 | 캐시샵 확장권으로 +50칸씩 |

> **[프로그래머 의견]** 무게 시스템은 유저가 직관적으로 이해하기 어려울 수 있다. 인벤토리 UI에 현재 무게/최대 무게 게이지를 상시 표시하고, 임계치(80%, 100%) 도달 시 경고를 띄워야 한다.
>
> **[디자이너 의견]** 무게 게이지는 인벤토리 하단에 바(bar) 형태로 표시. 80% 이상이면 노란색, 100% 이상이면 빨간색으로 변경.
>
> **[기획자 결정]** 수용.
