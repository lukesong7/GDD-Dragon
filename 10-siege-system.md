## 2. 기지 쟁탈전 규칙

### 2.1 진행 흐름

```
[신청 기간]
  공격 군단: 쟁탈전 신청 (최대 5군단)
  수비 군단: 현재 기지 사령관 군단 + 연합 군단
      ↓
[준비 기간 - 토요일 19:30~20:00]
  수비: 방어 시설 배치 (에너지 배리어, 방어 터렛, 지뢰)
  공격: 기지 외곽 집결
      ↓
[쟁탈 Phase 1 - 외곽 방어선 (30분)]
  공격측: 외곽 게이트 파괴 또는 실드 해킹
  수비측: 외곽 방어, 터렛 운용
      ↓
[쟁탈 Phase 2 - 중앙 사령부 (30분)]
  공격측: 중앙 사령부 진입, 제어실 돌입
  수비측: 사령부 사수, 최종 방어선
      ↓
[점령 판정]
  제어실의 메인 콘솔을 3분간 점유한 군단이 새 기지 사령관
  시간 종료 시 수비 군단 유지
```

### 2.2 공성 장비

| 장비 | 사용측 | 효과 | 조건 |
|------|--------|------|------|
| 공성 메카 | 공격 | 게이트 타격 (대폭 데미지) | 3인 탑승 운용 |
| EMP 폭탄 | 공격 | 범위 내 방어 터렛/드론 일시 무력화 (10초) | 설치 시 15초 소요 |
| 실드 브레이커 | 공격 | 에너지 배리어 관통/해제 | 조작 1인 + 지원 2인 |
| 방어 터렛 | 수비 | 자동 원거리 공격 | 미리 배치, 파괴 가능 |
| 플라즈마 방출기 | 수비 | 진입로 범위 에너지 데미지 | 방어선 위에서 사용 |
| 에너지 배리어 | 수비 | 통로 차단, 내구도 보유 | 미리 배치 |

> **[프로그래머 의견]** 공성 장비는 다수 플레이어가 동시에 인터랙션하는 객체다. 장비 조작 권한의 선점/해제 로직, 메카 탑승 시 플레이어 동기화는 네트워크 코드에서 가장 까다로운 부분이 된다. 상태 동기화를 장비 전용 프로토콜로 분리하는 것을 권장.
>
> **[디자이너 의견]** 공성 장비의 스케일감이 중요하다. 공성 메카가 게이트를 타격하며 금속 파편이 흩날리는 연출, EMP 폭탄의 전자기 파동 이펙트, 플라즈마 방출 시 열기 왜곡 효과 등이 기지 쟁탈전의 스펙터클을 만든다. 카메라 쉐이크와 함께.
>
> **[기획자 결정]** 모두 수용. 공성 장비 시스템은 프로토타입 단계에서 우선 검증해야 할 핵심 피처. 연출은 '과하지 않되 확실한' 수준을 목표로 한다.

## 3. 기지 사령관 보상 및 운영

### 3.1 기지 사령관 군단 보상

| 보상 | 내용 |
|------|------|
| **거래세 징수** | 기지 등급에 따라 해당 구역 거래세 2~5% 자동 징수 |
| **전용 NPC 상점** | 기지 내부 특별 NPC (희귀 소모품, 강화 모듈 판매) |
| **작전 구역 지배권** | 관할 작전 구역 드롭률 +10% (군단원) |
| **기지 사령관 칭호** | 기지 사령관 전용 칭호 + 외형 이펙트 |
| **기지 내부 시설** | 군단 전용 제작 터미널, 보급창고, 리스폰 지점 |

### 3.2 거래세 관리

- 거래세는 자동 징수되어 **기지 금고**에 축적
- 기지 금고에서 인출 가능: 사령관, 부사령관
- 용도: 방어 시설 수리/강화, 군단 운영비, 자유 분배
- **세율 조정**: 기지 사령관이 0~5% 범위 내에서 조정 가능

> **[기획자 의견]** 거래세 시스템은 기지를 보유할 '이유'를 만드는 핵심이다. 거래세가 너무 높으면 일반 유저 반발, 너무 낮으면 쟁탈 동기 부족. 최대 5% 상한은 경제 파괴를 방지하면서 의미 있는 수입을 보장한다.

## 4. 기지 쟁탈전 밸런스

### 4.1 공수 밸런스

- **수비 유리 원칙**: 기지의 지리적 이점으로 수비측이 기본적으로 유리
- **다수 공격 보정**: 공격 군단이 다수일 때 공격측 간 경쟁 발생 → 자연적 밸런스
- **공격측 보정**: 공격측 리스폰 지점을 기지 근처에 배치하여 이동 시간 단축

### 4.2 혼전 방지

- 공격 군단이 3개 이상이면 공격측끼리도 PvP 가능
- 제어실 점유는 한 군단만 가능 → 최종 국면에서 공격 군단 간 경쟁

> **[기획자 의견]** 공격 군단 간 혼전이 기지 쟁탈전의 백미다. 수비를 뚫은 뒤 제어실을 차지하기 위한 '최후의 난전'이 전설적인 순간을 만든다. 이 혼돈을 의도적으로 설계한다.

## 5. 영토전 (소규모)

기지 쟁탈전과 별도로, 작전 구역 단위의 소규모 점령전.

### 5.1 영토전 규칙

| 항목 | 내용 |
|------|------|
| 개최 | 매일 21:00~22:00 |
| 대상 | 각 작전 구역의 제어 비콘 |
| 참가 | 군단 Lv.4 이상 |
| 점령 | 비콘을 2분간 해킹/점유 |
| 유지 | 다음 영토전까지 유지 |

### 5.2 영토 보유 효과

- 해당 작전 구역 드롭률 +10% (군단원)
- 비군단원 작전 구역 이용 시 드롭 크레딧의 5%를 관세로 자동 징수
- 영토 방어용 자동 경비 드론 5기 배치 가능

> **[프로그래머 의견]** 영토전은 매일 발생하므로 시스템 자동화가 중요. 영토전 시작/종료, 비콘 상태 관리, 보유 효과 적용/해제가 모두 자동으로 처리되어야 한다. 크론 스케줄러 기반으로 구현.
>
> **[기획자 결정]** 수용. 영토전은 코드 레벨에서 완전 자동화되며, 운영 개입 없이 매일 실행된다.
